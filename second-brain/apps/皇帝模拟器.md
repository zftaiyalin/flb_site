# 皇帝模拟器（天命 / 帝王命 / 朕的一生）

## 产品定位

AI 增强的系统型人生模拟游戏（非 AI 聊天产品）。
玩家扮演古代皇帝，通过治理国家、批阅奏折、平衡派系，走向不同人生命运与王朝结局。
核心：**"规则驱动 + AI 命运演出"**，让玩家产生"这一局没打好，我想再活一次"。

## MVP 功能

- 主界面 + 时间推进系统（【下一年】核心按钮）
- 国家数值系统：国库 / 民心 / 皇权 / 军权 / 世家势力 / 后宫稳定度 / 边疆稳定度 / 太子支持率
- 奏折批阅系统（亲自批阅 / 交由内阁 / 置之不理）
- AI 命运事件系统（每半年读取数值 + 历史摘要生成重大命运事件）
- 因果系统（玩家历史行为影响未来）
- 结局系统 + 谥号系统
- 动态氛围系统（盛世/平稳/动荡/崩盘/天命终局）
- 开局配置（治国节奏 / 王朝难度 / 开局国策 / AI 命运强度）
- 广告系统（激励广告 + 天降祥瑞救场广告）

## 已确定功能

**奏折系统设计：**
- 游戏主界面顶部奏折按钮 + 未处理数量显示
- 三种处理方式：亲自批阅（低风险）/ 交由内阁（中性随机）/ 置之不理（偏坏结果）
- 奏折品质：普通 / 重要 / 紧急（红色急报）
- 奏折与 AI 命运联动：长期忽视某类奏折 → 触发对应命运事件

**结局系统：**
- 结局原型：盛世明君 / 铁血暴君 / 傀儡皇帝 / 长生疯帝 / 亡国之君
- 谥号：文 / 景 / 武 / 宣 / 厉 / 炀 / 幽
- AI 负责演出结局（后世评价 / 史官评价 / 人生总结）
- 玩家可选：【生成完整史书】（触发激励广告）或【直接重开人生】

**广告设计（不破坏沉浸感）：**
- 核心广告位：结局后生成完整史书（高情绪节点）
- 救场广告：国库见底/民心崩盘时低概率触发【天降祥瑞】
- 主动资源广告：资源旁边皇家按钮，玩家主动点击

## 后续版本功能

- 多世界观扩展：修仙人生 / 豪门人生 / 星际人生 / 黑帮人生（复用同一套系统框架）
- AI 视频（长期方向）
- 联网 + 社区（验证核心循环成立后）
- 大型后宫系统
- 多朝代

## UI/设计方向

**整体风格：** "淡雅国风 + 皇家极简"

**参考：** 水墨山水 / 宣纸 / 皇家卷轴 / 编年史 / 古代档案馆

**禁止：** 页游风 / 金灿灿暴发户风格 / MMORPG 宫廷 UI / 卡通化 / 二次元

**动画：** 山水漂浮 / 云雾移动 / 玉玺落印 / 墨水扩散 / 烛火粒子 / 卷轴展开

**国家状态动态氛围：**
- 盛世：青绿山水 + 云海 + 古琴笛子
- 动荡：乌云风沙 + 烛火摇晃 + 紧张鼓点
- 崩盘：暴雨雷电 + 黑红色调 + 战鼓低频
- 天命终局：大雪落叶 + 皇帝背影 + 孤独古琴

## 技术方案

- Flutter App（单机，MVP 完全本地化）
- LLM API（AI 命运事件 + 结局演出 + 史书生成）
- 不需要后台服务器（MVP 阶段）
- 订阅：RevenueCat
- 广告：AdMob

## 商业模式

- 激励广告：生成完整史书 + 天降祥瑞救场
- 主动资源广告
- 订阅（解锁高级命运强度、更多结局原型、无广告）

## 风险与问题

- AI 生成稳定性：需要有限命运模板约束 AI，防止世界观崩坏
- AI token 成本：每半年生成一次命运事件，需要控制频率和 prompt 长度
- 核心验证指标不是收入，而是"重开率"和"沉浸感"
- 长期扩展性：框架能否真正复用到修仙/豪门等世界观
